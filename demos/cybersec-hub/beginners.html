What is Cybersecurity?

Cybersecurity is the practice of protecting computers, networks, programs, and data from digital attacks, damage, or unauthorized access. It encompasses a wide range of technologies, processes, and practices designed to defend against an ever-evolving landscape of threats.

The CIA Triad

The three core principles that underpin all of information security:

C
Confidentiality
Only authorised users can access information. Achieved through encryption, access controls, and authentication.
I
Integrity
Information is accurate and hasn't been tampered with. Achieved through hashing, digital signatures, and checksums.
A
Availability
Systems and data are accessible when needed. Protected through redundancy, backups, and DDoS mitigation.

Essential Knowledge

Networking Fundamentals

You cannot do cybersecurity without understanding networking. These concepts are non-negotiable.

🌐

TCP/IP & OSI Model

The foundation of all network communication. Understand the 7 OSI layers (Physical to Application) and how TCP/UDP differ. Know common ports: 80 (HTTP), 443 (HTTPS), 22 (SSH), 25 (SMTP), 53 (DNS), 3389 (RDP).

📡

IP Addressing & Subnetting

IPv4 vs IPv6, public vs private addresses, CIDR notation (/24, /16), subnetting. Being able to subnet in your head impresses interviewers and is essential for network-based attacks and defences.

🔗

DNS & DHCP

DNS translates domain names to IP addresses. DHCP assigns IP addresses dynamically. Attackers abuse both — DNS poisoning, DHCP starvation. Understanding these protocols is critical.

🔥

Firewalls & NAT

Firewalls control traffic flow based on rules. Stateful vs stateless. NAT allows private networks to reach the internet. Understanding firewall rules is essential for both defence and evasion.

🌍

HTTP/HTTPS

How the web works — request/response cycle, HTTP methods (GET, POST, PUT, DELETE), status codes, cookies, sessions, and headers. Essential for web application security.

🔐

Encryption Basics

Symmetric (AES) vs asymmetric (RSA) encryption. Hashing (MD5, SHA-256) — not reversible. PKI, TLS/SSL certificates, and how HTTPS protects data in transit.

Threat Landscape

Common Attack Types

Every cybersecurity professional must understand the threats they're defending against.

🎣 Phishing
Deceptive emails/messages tricking users into revealing credentials or downloading malware. Accounts for 90%+ of data breaches. Spear-phishing targets specific individuals.
🦠 Malware
Malicious software: viruses (self-replicating), worms (network-spreading), Trojans (disguised as legitimate), ransomware (encrypts files for ransom), spyware, adware, rootkits.
🔓 SQL Injection
Inserting malicious SQL code into web forms to manipulate databases. Allows attackers to extract, modify, or delete data. One of the oldest and most common web attacks.
📜 XSS
Cross-Site Scripting — injecting malicious scripts into web pages viewed by other users. Can steal cookies, redirect users, or deface websites. Reflected, Stored, and DOM-based variants.
🤝 Man-in-the-Middle
Attacker intercepts communication between two parties. ARP poisoning, SSL stripping, rogue Wi-Fi access points. HTTPS mitigates many MITM attacks.
💥 DoS/DDoS
Denial of Service — overwhelming a system with traffic to make it unavailable. Distributed DoS uses botnets of compromised machines. Can take down websites and services.
🔒 Ransomware
Malware that encrypts victim's files, demanding payment for decryption keys. Major incidents include WannaCry, NotPetya, and REvil. Backup strategy is the primary defence.
👤 Social Engineering
Manipulating people rather than systems. Pretexting, impersonation, tailgating (physical), baiting. The human element is the hardest to secure.
Your Roadmap

10 Steps to Your First Cyber Job

A realistic, proven path from complete beginner to employed security professional.

1

Learn Networking Basics (4–8 weeks)

Professor Messer's free CompTIA Network+ course on YouTube. Understand subnetting, TCP/IP, DNS, HTTP. Don't skip this — it's the foundation everything else builds on.

2

Get Comfortable with Linux (4–6 weeks)

Install Ubuntu or Kali Linux in a VM (VirtualBox is free). Learn the command line — navigation, file permissions, user management, bash scripting. OverTheWire: Bandit is excellent practice.

3

Learn Windows Administration (2–4 weeks)

Most enterprise environments run Windows. Understand Active Directory, Group Policy, PowerShell, Windows Event Logs, and registry structure. Critical for both red and blue team work.

4

Earn CompTIA Security+ (2–4 months)

The industry's baseline cert. Opens doors to entry-level roles, required for many government/DoD positions. Professor Messer's free course + Darril Gibson's book. Costs ~$400 to take.

5

Start CTF Practice (ongoing)

Join TryHackMe — start with "Pre-Security" then "SOC Level 1" or "Jr Penetration Tester" paths. Practice makes perfect, and CTF experience looks excellent on resumes.

6

Learn a Scripting Language (2–4 months)

Python is the best choice for cybersecurity. Automate tasks, write simple tools, understand how exploits work. Codecademy or Automate the Boring Stuff (free) are great starting points.

7

Build a Home Lab

Virtualise an attack lab — a Kali attacker VM, a Windows Server victim, a vulnerable Linux machine (Metasploitable). Practice safely in isolation. Document everything for your portfolio.

8

Choose a Specialisation

SOC/Blue Team: CySA+, study SIEM (Splunk free training), and practice on LetsDefend. Pentesting: eJPT cert, HackTheBox, study for OSCP. Your choice shapes your next steps.

9

Build Your Portfolio & Network

Write CTF write-ups on a blog (GitHub Pages is free). Contribute to open source security tools. Get on LinkedIn, attend DEF CON / BSides meetups. Join Discord communities like TryHackMe's.

10

Apply for Entry-Level Roles

Target: SOC Analyst Tier 1, IT Helpdesk with security focus, Junior Penetration Tester, Security Analyst intern. Tailor your resume, prep for technical interviews, and leverage your portfolio.

Hands-On Learning

Building Your Home Lab

💻 Minimum Requirements

  • PC/Laptop: 16GB RAM (8GB minimum), modern CPU, 500GB+ SSD
  • Hypervisor: VirtualBox (free) or VMware Workstation (paid)
  • Attacker VM: Kali Linux — has 600+ pre-installed security tools
  • Victim VMs: Metasploitable 2/3, DVWA, VulnHub machines
  • Windows VMs: Windows Server eval (free from Microsoft)
  • Networking: Host-only adapter for isolated attack environment

🎯 What to Practice

  • Network scanning with Nmap — find hosts, ports, services
  • Web app attacks on DVWA (SQL injection, XSS, file upload)
  • Metasploit exploitation on Metasploitable
  • Wireshark packet capture and analysis
  • Set up a SIEM (Security Onion or Splunk free) — ingest logs
  • Active Directory lab (2x Windows Server) — Kerberoasting, BloodHound
  • Write up every exercise for your portfolio
