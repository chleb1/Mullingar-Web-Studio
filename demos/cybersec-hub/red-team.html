Foundation

What is Red Teaming?

Definition

Red teaming is a full-scope, adversarial simulation of real-world attacks against an organisation's people, processes, and technology. Unlike penetration testing, red team engagements are longer (weeks to months), stealthier, and goal-oriented — the aim is to reach a defined objective (e.g., access financial data) without being detected.

Red Team vs. Penetration Testing

  • Pentest: scoped, time-boxed, finds all vulnerabilities
  • Red Team: objective-based, stealth-focused, adversary simulation
  • Red Team tests the blue team's detection capability
  • Red Team engagements often go weeks or months

⚠️ Legal & Ethical Boundary

Red teaming is only legal with explicit written permission (Rules of Engagement). Unauthorized testing is a criminal offense in virtually every jurisdiction. Always have a signed Statement of Work before touching any system.

Core Roles

  • Red Team Lead — Plans and directs the engagement
  • Penetration Tester — Executes technical attacks
  • Social Engineer — Phishing, vishing, physical intrusion
  • Malware Developer — Custom implants and evasion
Methodology

The Attack Lifecycle

Red teams follow a structured methodology mirroring real-world threat actors.

1

Reconnaissance (OSINT)

Passive and active intelligence gathering — employee data from LinkedIn, DNS enumeration, Shodan searches, leaked credentials, domain information, and technology fingerprinting. Tools: Maltego, theHarvester, Recon-ng, OSINT Framework, SpiderFoot.

2

Scanning & Enumeration

Active discovery of live hosts, open ports, running services, OS versions, and misconfigurations. Tools: Nmap, Masscan, Nikto, Gobuster, Enum4linux, CrackMapExec, Nessus.

3

Initial Access

Gaining a foothold using phishing emails, exploiting public-facing vulnerabilities, password spraying, supply chain attacks, or physical access. Tools: Gophish, Metasploit, SQLmap, Evilginx2.

4

Privilege Escalation

Escalating from low-privilege user to admin/SYSTEM/root using kernel exploits, misconfigurations, credential abuse, or service exploits. Tools: WinPEAS, LinPEAS, BeRoot, PowerUp, Mimikatz.

5

Lateral Movement

Moving through the network to reach target systems. Pass-the-Hash, Pass-the-Ticket, Kerberoasting, BloodHound for AD path mapping. Tools: BloodHound, Impacket, Evil-WinRM, CrackMapExec.

6

Persistence & C2

Establishing persistent access via scheduled tasks, registry run keys, web shells, or implants. Maintaining C2 communication using Cobalt Strike, Sliver, Havoc, or Covenant. Tools: Cobalt Strike, Sliver, Havoc Framework.

7

Objective Completion & Exfiltration

Reaching the defined goal: accessing sensitive data, compromising critical infrastructure, or demonstrating domain dominance. Documenting every step with evidence.

8

Reporting & Remediation

Comprehensive written report covering attack path, evidence, business impact, and prioritised remediation recommendations. Debrief with the blue team (purple team exercise).

Knowledge

Core Concepts

📧

Social Engineering

Phishing, spear-phishing, vishing (voice), smishing (SMS), pretexting, and physical intrusion. The human is often the weakest link. Frameworks: SET (Social Engineering Toolkit).

🔐

Active Directory Attacks

Kerberoasting, AS-REP Roasting, DCSync, Pass-the-Hash, Golden Ticket, Silver Ticket, ACL Abuse. AD attacks are central to most enterprise red team engagements.

🌐

Web Application Attacks

SQLi, XSS, SSRF, XXE, IDOR, RCE, authentication bypass, JWT attacks, OAuth flaws, API abuse. OWASP Top 10 is the essential reference.

📡

Network Attacks

ARP spoofing, MITM, LLMNR/NBT-NS poisoning, DNS hijacking, Responder attacks, SMB relay. Tools: Responder, Ettercap, Bettercap.

🔓

Password Attacks

Brute force, dictionary attacks, credential stuffing, rainbow tables, hash cracking. Tools: Hashcat (GPU-accelerated), John the Ripper, Hydra, Medusa.

☁️

Cloud Attacks

S3 bucket misconfigs, IAM privilege escalation, SSRF for metadata, exposed keys, insecure Lambda functions. Tools: Pacu, ScoutSuite, CloudFox, Prowler.
