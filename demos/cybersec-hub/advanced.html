Malware Analysis

Analysis Approaches

Static Analysis

Examining malware without executing it. File hashing, string extraction, import table analysis, disassembly with IDA Pro or Ghidra. Reveals capabilities, IOCs, and code structure.

Dynamic Analysis

Running malware in a controlled sandbox environment. Observe file system changes, network connections, registry modifications, and process behaviour. Tools: Cuckoo Sandbox, ANY.RUN, FlareVM.

Code Reversing

Decompiling, disassembling, and debugging to fully understand what malware does. x86/x64 Assembly knowledge required. Tools: IDA Pro, Ghidra, x64dbg, dnSpy (.NET).

Discipline

Reverse Engineering

The art of understanding how software works without access to source code — fundamental for exploit development and malware analysis.

📖

x86/x64 Assembly

The language of the CPU. Understanding registers (RAX, RBX, RSP, RBP), stack frames, calling conventions, and common instruction patterns (MOV, PUSH, POP, CALL, JMP, CMP, JE). Non-negotiable for serious RE.

🔬

Decompilation vs Disassembly

Disassembly converts machine code to assembly. Decompilation lifts to pseudo-C. Modern tools like Ghidra and IDA provide both. For .NET/Java, tools like dnSpy or jadx can recover near-source code.

🐛

Anti-Analysis Techniques

Malware resists analysis through packing, obfuscation, anti-debugging (IsDebuggerPresent, timing checks), VM/sandbox detection, and code encryption. Learn to identify and defeat each technique.

🔓

License Cracking (for Research)

Analysing software protection mechanisms — serial number algorithms, licence checks, packing. Excellent for building RE skills on crackme challenges (crackmes.one). Never apply to production software.

💻

Firmware Analysis

Extracting and reversing embedded device firmware. Binwalk for extraction, QEMU for emulation. Critical for IoT security research. Growing field as IoT devices proliferate.

📱

Mobile App Reversing

Android APKs (jadx, apktool, Frida) and iOS IPAs (class-dump, Hopper). Mobile banking trojans, spyware, and certificate pinning bypass are key areas of mobile security research.

Discipline

Exploit Development

Memory Corruption Vulnerabilities

  • Buffer Overflow (Stack-based): Overwriting return address to redirect execution
  • Heap Overflow: Corrupting heap metadata for code execution
  • Use-After-Free: Accessing freed memory — UAFs are a major class of browser bugs
  • Format String: Exploiting %n/%s format specifiers for arbitrary write
  • Integer Overflow: Wrapping integer values to cause allocation errors

Exploitation Mitigations (and bypasses)

  • ASLR — Address Space Layout Randomisation → info leak bypass
  • DEP/NX — Data Execution Prevention → ROP chains
  • Stack Canaries — Detect stack overflows → info leak bypass
  • CFG — Control Flow Guard → still active research area
  • RELRO — Relocation Read-Only → GOT overwrite difficulty

Intelligence

APTs & Threat Intelligence

🎭

Nation-State Actors

APT28/Fancy Bear (Russia), APT41 (China), Lazarus Group (North Korea), Charming Kitten (Iran). Study their TTPs using MITRE ATT&CK groups. Each has distinct tooling, targets, and operational patterns.

📡

Threat Intelligence Lifecycle

Planning → Collection → Processing → Analysis → Dissemination → Feedback. Structured intelligence production for executive, tactical, and operational consumers. STIX/TAXII for sharing.

🔎

IOC Hunting

Indicators of Compromise — IP addresses, domains, file hashes, mutexes, registry keys. Tools: VirusTotal, Shodan, Censys, URLhaus, Greynoise. Balance breadth (IoCs) with TTPs (longer shelf life).
